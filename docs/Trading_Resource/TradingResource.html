Bet Activity Report:

https://trading-tool.digitalsportstech.com/#!/trading-reports/bet-activity

  • Real-time betting activity from our clients
  • Always have open on one screen when trading
  • Can filter by sport if only covering a specific sport for trading
  • When busy, filter by Focus Mode in the top right. Make sure it’s blue

  • Red user names mean they are winning users on our platform and should be respected. If the name is bold, that means the user has a note on his profile. Click the name to review if needed.
  • Yellow row highlights indicate a large bet (over $250)
  • Red row highlights indicate a sharp bet. We should adjust our line/price when we see this.
  • Book exposure column indicates how much we are set to win/lose if that market wins. Red indicates loss.

O/U Trading Report:

https://trading-tool.digitalsportstech.com/#!/trading-reports/ou-trading

  • This is where we see our O/U markets and make price and line adjustments
  • We trade based on winning users and sharps. The more winning users on a side, the more we need to be shortening price or moving line in that direction to get betting on the other side.
  • We are not trying to get balanced activity. We are trying to move money away from winning and sharp sides.
  • Comps section is like our Don Best area where we can see comp lines/prices move in real-time. We use them as a reference but we should feel comfortable moving outside the range if our activity demands it.
  • If Auto-trading button is on (green), this means the market pricing is just copying the comps. If we need to manually intervene due to betting activity, we turn off auto-trading (red) and make manual adjustments.
  • Active button indicates the O/U market is active. Disabling this will turn off O/U for the player, but the player stays active for single-stat betting. We rarely use this.
  • Click on the player’s name to see the betting activity on him.
  • Make sure the Auto Reload button is on (blue) to get real-time updates. It shuts off temporarily when clicking in cells but should come back on after 5 seconds once you click out. If it stays off, do a hard refresh of the page

Bet Exposure Report:

https://trading-tool.digitalsportstech.com/#!/trading-reports/bet-exposure-report

  • This is where we see exposure on non-over/under markets.
  • Single-Stat (SIN) bets are typically higher odds such as a player to score 35+ points.
  • We should be watching for any markets that seem to be overly popular compared to others in terms of total bets, total wagered or total exposure.

Settlement Screening Tool:

https://trading-tool.digitalsportstech.com/#!/settlement-result

  • This is where we settle Soccer Boxscores manually if needed
  • We wait until 15 minutes after the game has ended to start settling boxes
  • Select the league and game from the dropdown menu.
  • Click pull latest data, wait for the data to populate the input boxes
  • Click Save
  • Sort by Minutes played
  • Check if the data and box look okay. We are looking to make sure every player is correctly marked as played and started and has stats. Everyone that played should be green and everyone that started should be green.
  • If a player has a search icon next to their name, that means the player isn’t mapped properly and the stats didn’t scrape. We need to click the icon, find the corresponding player and click him. This should map the player and new stats should populate. Once this is done, click save again and wait for it to save. Ask for help if you haven’t done this before.
  • If you need to manually enter a player’s stats, we use Whoscored.com as the source. Find the game, click into the game center and then player statistics. Ask for help if you haven’t done this before.
  • Once everything looks done, click start verification. You should get a success message in the top right.
  • Mark the boxscore as done in green with your name in the schedule
