SST Settlement Steps

Soccer

Steps for boxscore settlement:

  1. We wait until 30 minutes after the game has ended to start settling boxes
  2. Select league and game
  3. Click [Pull DSG Results] and wait for the data to populate the input boxes[a]
  4. Click [Save] once stats appear. It takes under a minute for a save success message to appear
  5. Sort by Minutes played
  6. Check if the data and box look okay. We are looking to make sure every player is correctly marked as played and started and has stats. Everyone that played should be green and everyone that started should be green.
  7. If a player has a search icon next to their name, that means the player isn’t mapped properly and the stats didn’t scrape. We need to click the icon, find the corresponding player and click him. This should map the player and new stats should populate.
  1. Once this is done, click [Save] again and wait for it to save. Ask for help if you haven’t done this before.
  1. If you need to manually enter a player’s stats, we use Whoscored.com as the source. Find the game, click into the game center and then player statistics. Ask for help if you haven’t done this before.
  2. Once everything looks done, click [Start Verification]. You should get a success message in the top right.

Some bugs to be aware of:

  • Player's that aren't mapped will show as not played/started. You need to toggle those correctly and save that as well.

Player mapping

  • When you pull latest data you might notice plus icon next to player name, this means that player was not found in data providers API, by clicking on plus icon, you will see a modal window with player list, it will contain all players for that game from provider.
  • Select a player from the list which you want to map that data to and data should appear for that player in the table
  • Note that sometimes plus icon is shown because that player simply wasn’t available in provider’s API, you can verify by clicking plus icon and see if that player appears in the list, in case where he does not, you can either wait a bit and pull latest data again OR input data manually into the table  

[a]If you don't see overrides in input boxes, including for Mins played, then you know the data hasn't been pulled from DSG
